# Security Policy

## Supported Versions

Security updates are provided for the latest stable release on the `main` branch.

Older releases may not receive security updates. Please upgrade to the latest version before reporting a vulnerability.

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please **do not** create a public GitHub issue.

Instead, report it privately by contacting me on X (formerly Twitter):

**@kunalghosh58540**

When reporting a vulnerability, please include:

* A clear description of the issue.
* Steps to reproduce the vulnerability.
* The affected version, commit, or branch.
* Any proof of concept, logs, or screenshots (if applicable).
* Suggestions for mitigation or a potential fix (optional).

### What to Expect

* I will acknowledge your report as soon as reasonably possible.
* I will investigate the issue and determine its impact.
* If the report is confirmed, I will work on a fix and release it as soon as practical.
* You may be asked for additional information during the investigation.

## Responsible Disclosure

Please do not publicly disclose the vulnerability until a fix has been released or we have agreed on a disclosure timeline.

Thank you for helping make this project more secure.
